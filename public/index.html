URL-shortener

This is a URL shortener developed in Node.js and using MongoDB as a database.

It provides a basic service of storing an URL (up to 300 characters) and saves it in our database, giving you only a 24 character hex string that serves as an ID. You can later get the original URL back by providing another API call with that ID.

How it works?

There are 2 API endpoints currently.

  • /shorten/<url> - shortens the URL and returns a JSON response - either an error if something went wrong, or a regular response object (specified below). If this URL was already shortened before then it returns the previous ID of the shortened URL
  • /url/<id> - redirects to a url that corresponds to this id. When no url was found displays a JSON error object.

Response objects

There are 2 types of reponse objects returned from the API.

  • Regular response

    Contains an _id field that holds a 24 character long hex string that identifies that url, and an url field that holds the original URL that was to be shortened

    {"_id":"56b12102bcdcb258201c5235","url":"hello-world"}

  • Error response

    Contains only one field - error, which contents specify what happened exactly.

    {"error":"this url is too long"}
